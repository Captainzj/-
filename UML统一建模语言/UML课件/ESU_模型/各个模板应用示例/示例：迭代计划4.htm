T1 和 T2 迭代计划

1. 目标

    本迭代计划说明了产品化阶段的最初两次迭代的详细计划。

    在产品化阶段的第一次迭代 (T1) 中，封装和安装课程注册系统的 R1.0 版本。除提交和查看学生分数功能外，这个发布版几乎包括了课程注册系统的所有关键功能。

    在产品化阶段的第二次迭代 (T2) 中，封装和安装 R2.0 版本。此发布版包括了前景文档 [1] 中所定义的所有关键功能。

2. 范围

    此迭代计划适用于 Wylie College 信息系统部门为该校开发的课程注册系统项目。本文档供项目经理、部署经理和项目团队使用。

4. 计划

当开发团队制定出 R1.0 软件基线后，就可以开始第一次迭代 (T1) 了。软件将被封装，并制成分发介质。产品化团队将编写用户手册，并把软件安装到 Wylie College 服务器和校园内所选的客户 PC 机。产品化团队负责执行用户验收测试并取得 Wylie College 代表的认可。

此外，还要将客户机程序安装到一个 FTP 站点上，这样任何一台具有有效 ID 和 Internet 访问权限的客户 PC 机都可以下载客户机部分的程序。

在安装软件以后，产品化团队为 Wylie College 的注册员和管理人员提供培训和技术支持。

当开发团队制定出 R2.0 软件基线时，就可以开始第二次迭代 (T2) 了。R2.0 发布版的准备活动与上述的 R1.0 活动相同，只是 R2.0 还必须转换和导入 R1.0 的数据。

构成发布版 1.0 的主要用例有：

    • 登录
    • 结束注册
    • 课程注册
    • 课程目录数据库的接口
    • 财务系统的接口
    • 维护学生信息
    • 维护教授信息

构成发布版 2.0 的主要用例有：

    • 提交学生成绩
    • 查看成绩
    • 选择要讲授的课程

7. 评估标准

T1 和 T2 迭代的主要目标是安装 R1.0 和 R2.0 并取得用户对软件的验收。

另外，在完成 T1 和 T2 迭代以前，还必须符合下列标准：

  • 发布版 R1.0 和 R2.0 不能有任何非常严重或很严重的缺陷。
  • 在秋季开学学生开始注册前 3 周，R2.0 必须已经安装好并且能够运行。
  • R1.0 和 R2.0 必须通过用户验收，并且得到 Wylie College 代表的正式认可。
  • 必须完成用户培训。
  • 必须建好提供产品支持的 Web 站点。
  • 必须印好用户手册，而且可以让用户从 IT 部门或者 Wylie College 书店获得这些手册。
  • 必须建立实时的为用户提供支持的流程和资源。
