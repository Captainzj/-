维护教授信息用例
  1. 简要说明

    本用例允许注册员维护注册系统中的教授信息。其中包括添加、修改和从系统中删除教授信息。

    本用例的主角是注册员。

  2. 事件流

    当注册员从主窗体中选择“维护教授信息”时，本用例就开始使用了。

    1. 基本流 — 添加教授信息
    1. 注册员选择“添加教授信息”。
    2. 系统会显示一张空白教授信息表。
    3. 注册员为教授输入以下信息： 姓名、出生日期、社会保障号、目前状况和所在系。
    4. 系统验证数据以确保数据格式正确，并按照指定的姓名来搜索已经在系统中存有记录的教授。如果数据有效，系统将创建一个新的教授信息并分配一个由系统生成的唯一 ID 号。该号码将显示出来，这样就可以在以后的系统使用中使用它。
    5. 每向系统添加一个教授信息，都要重复步骤 2-4。当注册员结束向系统中添加教授信息时，此用例结束。
    1. 备选流

      1. 修改教授信息

    1. 注册员选择“修改教授信息”。
    2. 系统会显示一张空白教授信息表。
    3. 注册员键入他（她）要修改的教授 ID 号。
    4. 系统检索该教授的信息并将信息显示在屏幕上。
    5. 注册员修改一个或多个教授信息字段：姓名、出生日期、社会保障号、目前状况和部门。
    6. 一旦修改结束，注册员选择“保存”。
    7. 系统更新该教授的信息。
    8. 注册员每修改一个教授信息，都要重复步骤 2-7。一旦编辑完成，本用例也就结束。

2. 删除教授信息

    1. 注册员选择“删除教授信息”。
    2. 系统会显示一张空白教授信息表。
    3. 注册员键入将被删除的教授 ID 号。
    4. 系统检索教授信息并将信息显示在表单中。
    5. 注册员选择“删除”。
    6. 系统会显示一个删除确认对话框以确定删除操作。
    7. 注册员选择“是”。
    8. 教授信息就被从系统中删除。
    9. 每删除一个教授信息，都要重复步骤 2-8。当注册员完成从系统中删除教授信息时，此用例结束。

3. 教授信息已经存在

        在“添加教授信息”子流程中，如果指定姓名的教授的信息已经存在，则将会显示一个“教授信息已经存在”的错误消息。注册员可以修改姓名、选择创建另一个同名教授的信息或者取消本次操作，此时用例结束。

4. 未发现教授信息

在“修改教授信息”或“删除教授信息”子流程中，如果指定 ID 号码的教授的信息不存在，系统将会显示一个“未发现教授信息”的错误消息。注册员可以键入另一个 ID 号码或者取消本次操作，此时用例结束。

3. 特殊需求

    没有和本用例有关的特殊需求。

4. 前置条件

  1. 登录

    在本用例开始前，注册员要登录到系统。

5. 后置条件

    没有和本用例有关的后置条件。

没有和本用例有关的扩展点。
